# 2026-03-09 - 股票机器人意图解析优化

## 讨论概要

优化了股票分析机器人的意图解析模块，解决 LLM 超时问题和意图识别错误。

## 主要进度

### 1. 系统提示词简化

**问题：** 原来 41K 字符 ≈ 10K token，本地 LLM 处理超过 60 秒超时

**解决：** 简化到 ~2K 字符 ≈ 500 token
- 只保留约 30 条常见股票示例
- 其他股票靠 `resolve_stock_code` 部分匹配查找

**文件：**
- `/home/ubuntu/.openclaw/workspace/stock_crawler/llm_system_prompt.txt`
- `/home/ubuntu/.openclaw/workspace/stock_crawler/llm_tools.json`

### 2. 添加 search_stock 工具

**目的：** 让 LLM 可以查询股票名称

**工作流程：**
```
用户输入: "兆豐股價"
    ↓
步骤 1: 预处理映射（约 100 条常见股票）
    → "兆豐" 不在映射表
    ↓
步骤 2: 规则解析
    → intent=unknown, confidence=0.5
    ↓
步骤 3: LLM 解析
    → 返回: {intent: 'search_stock', query: '兆豐'}
    ↓
步骤 4: search_stock 处理
    → 调用 resolve_stock_code('兆豐')
    → 返回: ('2886', '兆豐金')
    ↓
步骤 5: 返回最终结果
    → {intent: 'get_stock_price', stock_code: '2886'}
```

### 3. 统一股票名称验证流程

**问题：** "分析一下大成" 被错误解析成 "一下大成"

**解决：** 
1. 规则解析：清理股票名称中的无用词（"一下"、"的"、"這個"等）
2. 所有涉及股票名称的输入都先用 `resolve_stock_code` 验证
3. 简化流程，减少重复代码

**测试结果：**
| 输入 | 意图 | 股票代码 | 来源 |
|------|------|---------|------|

| 分析 大成 | brief_analysis | 1210 | search ✅ |
| 大成股價 | get_stock_price | 1210 | search ✅ |
| 分析一下台積電 | brief_analysis | 2330 | search ✅ |
| 台積電股價 | get_stock_price | 2330 | search ✅ |

## 意图解析流程重构

### 统一流程

```
用户输入
    ↓
步骤 1: 预处理映射（约 100 条常见股票）
    → Y 找到股號和 intent → 返回结果
    ↓ N
步骤 2: 规则解析
    → Y 找到股號和 intent → 返回结果
    ↓ N
步骤 3: LLM 解析（返回 search_stock + 股名）
    ↓
步骤 4: search_stock 处理
    → 调用 resolve_stock_code('股名')
    → 返回: ('股號', '股名')
    ↓
步骤 5: 返回最终结果
    → {intent: 'get_stock_price', stock_code: '股號'}
```

### 文件更改

| 文件 | 更改 |
|------|------|
| `intent_parser.py` | 重构，统一流程 |
| `llm_system_prompt.txt` | 简化到 ~500 token，LLM 只返回 search_stock |

### LLM 角色变更

之前：LLM 直接返回意图（get_stock_price、brief_analysis 等）
现在：LLM 只返回 `search_stock` + 股名 + 原始意图，由 `resolve_stock_code` 验证

所有测试用例通过，预处理映射覆盖常见股票，规则解析处理边缘案例。

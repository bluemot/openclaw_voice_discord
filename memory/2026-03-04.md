## Bug Fix: Minified JS Chunking (2026-03-04 09:53 UTC)

### Problem
- Remote Ollama embedding API returned 500 error for certain files
- Root cause: `lunr.min.js` was a single line with 33,842 characters
- The `_chunk_fn` function only chunked by line count, not character length
- bge-m3 context length is 8192 tokens, exceeded by large single-line files

### Solution
Modified `/home/ubuntu/workspace/crag/codebase_rag/main.py`:
- Added `max_chars=8000` parameter to `_chunk_fn`
- For single-line files exceeding `max_chars`:
  1. First try to split by semicolons (`;`)
  2. If segments still too long, hard-truncate
- For multi-line chunks exceeding `max_chars`, recursively split

### Test Results
- `lunr.min.js` (33,842 chars, 1 line) → 5 chunks (each < 8000 chars) ✅
